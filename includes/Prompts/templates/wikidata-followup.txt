# SPARQL GRAMMAR REMINDERS
- Triple chaining: `;` (same subject), `,` (same subject + predicate).
- Property paths: `/` sequence, `^` inverse, `*` zero-or-more, `+` one-or-more, `?` zero-or-one, `|` alternative.
- Patterns: OPTIONAL { ... }, UNION, MINUS, FILTER (...), FILTER EXISTS / NOT EXISTS, BIND(expr AS ?v), VALUES ?v { ... }, subqueries.
- FILTER built-ins: STR, LANG, BOUND, IF, COALESCE, REGEX, CONTAINS, STRSTARTS, STRENDS, YEAR, MONTH, DAY, NOW, xsd:dateTime("...").
- Aggregates require GROUP BY for every non-aggregated select var; filter aggregates with HAVING.
- Modifier order after WHERE: GROUP BY → HAVING → ORDER BY ASC/DESC → OFFSET → LIMIT.
- Wikidata prefixes (already declared): wd:, wdt:, p:, ps:, pq:, psv:, pqv:, wikibase:, bd:, rdfs:, schema:, skos:, xsd:.

## Pitfalls
- `wdt:` cannot reach qualifiers — use `p:/ps:/pq:` for start date, end date, rank, point in time, etc.
- Date comparisons need typed literals: `FILTER(?d > "1900-01-01T00:00:00Z"^^xsd:dateTime)`.
- OPTIONAL leaves vars unbound on miss — guard with BOUND or COALESCE.
- Property paths cannot be used inside SERVICE wikibase:label.
- Aggregating label variables (e.g., `GROUP_CONCAT(?valueLabel)`) causes them to return as empty strings. To aggregate labels, you MUST resolve them inside a subquery BEFORE applying `GROUP BY` and `GROUP_CONCAT` in the outer query.

# QUERY RULES
1. SERVICE wikibase:label must be the LAST clause INSIDE the WHERE block, immediately before the closing `}`. Use `SERVICE wikibase:label { bd:serviceParam wikibase:language "[AUTO_LANGUAGE],{{lang}},en" }` and `?xLabel` variables to receive labels.
2. SELECT only. No PREFIX declarations. LIMIT default 10, max 50; LIMIT 1 for superlatives.
3. New item placeholders → `wd:PLACEHOLDER`; new property placeholders → `wdt:PLACEHOLDER` (or `p:`/`ps:`/`pq:` with the same placeholder for qualifiers).
4. Ordinal/superlative ("first", "last", "oldest", "most recent", "tallest", etc.) → ORDER BY ASC/DESC with LIMIT 1.
5. Statement qualifiers (start date, end date, rank, point in time): `?item p:PROP ?stmt . ?stmt ps:PROP wd:VALUE ; pq:QUALIFIER ?val .` — same PID across `wdt:`, `p:`, `ps:`, `pq:`.
6. Class-based searches → `wdt:P31/wdt:P279*` instead of `wdt:P31` alone.
